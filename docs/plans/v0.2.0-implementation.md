  - `verifier_inactive_unknown_hash` — 不存在的 hash → inactive

  - `POST /service-accounts` — 创建 + 校验 tenant_id 存在
  - `GET /service-accounts?tenant_id=` — 列表
- [ ] 4.5 创建 `crates/aspectus-server/src/routes/api_keys.rs`
  - `POST /api-keys` — 调用 `ApiKeyCreator` + 返回原文
  - `GET /api-keys?service_account_id=` — 列表（无 hash/原文）
  - `DELETE /api-keys/{id}` — 吊销 + 删 Redis 缓存
- [ ] 4.6 更新 `main.rs` — 路由注册（mgmt sub-router + introspect + middleware）
- [ ] 4.7 `cargo build -p aspectus-server` + 启动

### 验证

```bash
source .env && cargo run -p aspectus-server &

# Seed a service token first (via psql or Step 5 migration)
# Then:

# 1. Create tenant
curl -s -X POST :3100/tenants \
  -H "Authorization: Bearer aspectus-dev-pandaria-service-token" \
  -H "Content-Type: application/json" \
  -d '{"name":"acme"}' | jq .

# 2. Create SA
curl -s -X POST :3100/service-accounts \
  -H "Authorization: Bearer aspectus-dev-pandaria-service-token" \
  -H "Content-Type: application/json" \
  -d '{"tenant_id":"<id>","label":"ci"}' | jq .

# 3. Create API Key
curl -s -X POST :3100/api-keys \
  -H "Authorization: Bearer aspectus-dev-pandaria-service-token" \
  -H "Content-Type: application/json" \
  -d '{"service_account_id":"<id>","project":"pandaria","scopes":[]}' | jq .

# 4. Introspect
curl -s -X POST :3100/introspect \
  -H "Authorization: Bearer aspectus-dev-pandaria-service-token" \
  -d "token=<key>" | jq .
# → { "active": true, ... }

# 5. Revoke
curl -s -X DELETE :3100/api-keys/<id> \
  -H "Authorization: Bearer aspectus-dev-pandaria-service-token"
# → 204

# 6. Introspect again
curl ... → { "active": false }

# 7. Invalid Service Token
curl -s -X POST :3100/introspect \
  -H "Authorization: Bearer wrong" \
  -d "token=..." | jq .
# → 401
```
