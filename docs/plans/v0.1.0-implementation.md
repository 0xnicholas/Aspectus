## Step 3 — 域模型 (`aspectus-core`)

### 目标

所有 struct、enum、trait 定义就绪。无方法实现（或空 `todo!()`）。

> **原则**：此 Step 只做类型定义，不做任何 DB 查询、HTTP 处理、业务逻辑。所有 struct 加上 `#[derive(Debug, Clone, Serialize, Deserialize, FromRow)]` 即使 `FromRow` 在 v0.1.0 无实际查询——提前标注避免 v0.2.0 遗漏。

### 操作清单（按 spec §5 顺序，逐文件创建）

- [ ] 3.1 `error.rs` — `CoreError` enum（thiserror）
  ```rust
  use thiserror::Error;
  #[derive(Debug, Error)]
  pub enum CoreError {
      #[error("invalid project: {0}")]
      InvalidProject(String),
      #[error("invalid scope format: {0}")]
      InvalidScope(String),
      #[error("entity not found: {entity} id={id}")]
      NotFound { entity: &'static str, id: String },
      #[error("validation failed: {0}")]
      Validation(String),
      #[error("internal error: {0}")]
      Internal(String),
  }
  ```
- [ ] 3.2 `project.rs` — `Project` enum + Display + FromStr
- [ ] 3.3 `identity.rs` — `IdentityType` + `RoleType` enum
- [ ] 3.4 `tenant.rs` — `Tenant` struct
- [ ] 3.5 `service_account.rs` — `ServiceAccount` struct
- [ ] 3.6 `api_key.rs` — `ApiKey`, `CreatedApiKey`, `ApiKeyListItem` struct
- [ ] 3.7 `scope.rs` — `Scope` struct
- [ ] 3.8 `audit_log.rs` — `AuditLog` struct
- [ ] 3.9 `user.rs` — `User` struct（注意 `#[serde(skip)]` 在 password_hash 上）
- [ ] 3.10 `role.rs` — `Role` struct
- [ ] 3.11 `introspect.rs` — `IntrospectResponse` struct + `inactive()` 构造函数
- [ ] 3.12 `lib.rs` — 声明所有模块 `pub mod`

### 验证

```bash
cargo build -p aspectus-core
# → 0 errors, 0 warnings

# 确认所有模块可被外部引用：
cargo doc -p aspectus-core --no-deps
# → 文档生成无错误（浏览器打开用 --open，仅本地开发时加）
```

### 关键注意事项

- `password_hash` 字段必须 `#[serde(skip)]`：编译期阻止序列化泄漏
- `#[sqlx(type_name = "...")]` 中 DB enum 是小写蛇形，Rust enum variant 是 CamelCase——`rename_all = "lowercase"` 处理大部分，但 `ServiceAccount` 需显式 `#[sqlx(rename = "service_account")]`
- `chrono::DateTime<Utc>` 与 `sqlx` 的 `chrono` feature 配合自动映射到 `timestamptz`
- `IntrospectResponse` 全部字段 `Option` + `skip_serializing_if`，确保 `active: false` 时只序列化一个字段

### 检查点

```bash
cargo build -p aspectus-core  # 通过
cargo build -p aspectus-auth   # 通过（依赖 core，空 lib.rs）
cargo build -p aspectus-client # 通过（依赖 core，空 lib.rs）
```

## 常见问题

### Q: `cargo build` 报 `edition 2024 is unstable`

A: 需要 Rust >= 1.85。`rustup update stable`。

### Q: `sqlx migrate run` 报连接被拒

A: `docker compose up -d` 后等 3 秒让 PostgreSQL 就绪。或用 `docker compose ps` 确认 health 状态。

### Q: `cargo test` 报 testcontainers 找不到 Docker

A: testcontainers 需要本地 Docker daemon 运行。macOS: Docker Desktop 启动即可。CI 环境自带。

### Q: `psql` 命令不存在

A: `brew install libpq` (macOS) 或 `apt install postgresql-client` (Linux)。或者用 `docker compose exec postgres psql -U aspectus`。
