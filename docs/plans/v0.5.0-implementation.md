## Step 1 — Migration + Store

- [ ] Migration 006: `api_keys.user_id` + nullable SA + one_owner check
- [ ] Migration 007: `users_roles.role_type` check constraint 激活
- [ ] Migration 008: Role 种子数据 (4 roles) + roles_scopes 映射
- [ ] `UserStore` trait (create, get_by_id, list_by_tenant, set_suspended)
- [ ] `PgUserStore` 实现
- [ ] `AppState` 加 `user_store: Arc<PgUserStore>`

---

## Step 2 — PasswordHasher

- [ ] `PasswordHasher` struct (argon2id: salt_len=16, hash_len=32)
- [ ] `hash(password) → String`
- [ ] `verify(password, hash) → bool`
- [ ] 单元测试

---

## Step 3 — User Routes

- [ ] `POST /users`: 校验 → hash → insert → 查 `roles WHERE is_default=true AND type IN ('user','both')` → 自动分配 → 写 `user.created` 审计 → 201
- [ ] `GET /users/{id}`: 查用户 + 查 roles → 返回（不含 hash）
- [ ] `GET /users?tenant_id=`: 列出用户
- [ ] `PUT /users/{id}/suspend`: `{ "suspended": true/false }` → 写 `user.suspended` 审计

---

## Step 4 — Role Routes + ScopeExpander

- [ ] `GET /roles`: 列出所有 Role
- [ ] `POST /users/{id}/roles`: `{ "role_id": "..." }` → check role_type → insert → 写 `role.assigned` 审计 → 201
- [ ] `DELETE /users/{id}/roles/{role_id}`: 移除 → 写 `role.removed` 审计 → 204
- [ ] `ScopeExpander`（放在 `aspectus-server`）: user_id → roles → roles_scopes → scopes → join → 去重 → 空格字符串

---

## Step 5 — api_keys owner_type

- [ ] `POST /api-keys` 请求体加 `owner_type` + `owner_id`（**同时兼容旧 `service_account_id` 字段**）
- [ ] Handler 根据 `owner_type` 查 users/service_accounts 获取 tenant_id
- [ ] `ApiKeyCreator` 扩展：接受 `owner: IdentityOwner`
- [ ] 对 User: scope 从 Role 展开（通过 `ScopeExpander`，放在 `aspectus-server`）；对 SA: 现有行为不变

---

## Step 6 — /introspect + 测试

- [ ] `/introspect` handler: verifier 返回基本响应 → 若 User token 则调 `ScopeExpander` 覆盖 scope 字段
- [ ] 集成测试：User CRUD、Role assign、role_type 约束、/introspect user、SA 回归
