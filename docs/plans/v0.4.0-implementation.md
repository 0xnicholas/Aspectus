# v0.4.0 — JWT + Opaque Token 实现计划

> 关联 spec：[../specs/v0.4.0-jwt-opaque-token.md](../specs/v0.4.0-jwt-opaque-token.md)

---

## 执行顺序

```
Step 1 ──→ Step 2 ──→ Step 3 ──→ Step 4 ──→ Step 5
 deps +     JWT        Token      Routes +    Test
 migration  sign/verify Verifier   JWKS
```

| Step | 内容 | 预估 |
|------|------|:--:|
| [1](#step-1--依赖--migration) | 依赖 + `token_type` 列 + `TokenFormat` enum + `IntrospectResponse.token_format` | 30 min |
| [2](#step-2--jwt-签发验证) | `JwtSigner` + `JwtVerifier` + 单元测试 | 1.5 h |
| [3](#step-3--tokenverifier-dispatch) | `TokenVerifier` 统一入口 + `OpaqueVerifier` | 1 h |
| [4](#step-4--路由) | `POST /token` + `POST /token/revoke` + `/.well-known/jwks.json` | 1 h |
| [5](#step-5--集成测试) | 集成测试 + 端到端验证 | 1 h |

---

## Step 1 — 依赖 + Migration

- [ ] `aspectus-auth/Cargo.toml` 加 `jsonwebtoken = "9"`, `rsa = "0.9"`
- [ ] `aspectus-server/Cargo.toml` 加 `jsonwebtoken.workspace = true`
- [ ] Migration 006: `ALTER TABLE api_keys ADD COLUMN token_type`
- [ ] `aspectus-core`: 加 `TokenFormat` enum (`api_key`/`jwt`/`opaque`)
- [ ] `IntrospectResponse` 加 `token_format: Option<String>`

---

## Step 2 — JWT 签发/验证

- [ ] `JwtSigner`: RS256 密钥管理（env 加载或自动生成）、`sign(sa_id, tenant_id, project, scopes, exp)` → JWT
- [ ] `JwtVerifier`: `verify(token)` → 验签 → exp → Redis 吊销集 → `IntrospectResponse`
- [ ] `POST /token/revoke` 辅助：提取 jti + exp → `SETEX jwt_revoked:{jti}`
- [ ] 单元测试：sign/verify roundtrip、过期、错误签名、吊销

---

## Step 3 — TokenVerifier Dispatch

- [ ] `TokenVerifier` struct 封装 `ApiKeyVerifier` + `JwtVerifier` + `OpaqueVerifier`
- [ ] `OpaqueVerifier`: 复用 `ApiKeyVerifier` 路径，使用 `ot_` 前缀
- [ ] `/introspect` handler 改为使用 `TokenVerifier`

---

## Step 4 — 路由

- [ ] `POST /token`: 验证 Service Token → ApiKeyVerifier 验证 `client_secret` → 签发 JWT/Opaque
- [ ] `POST /token/revoke`: 解码提取 jti → Redis SETEX
- [ ] `GET /.well-known/jwks.json`: 返回 public key JWKS
- [ ] 注册路由到 main.rs

---

## Step 5 — 集成测试

- [ ] JWT 全流程：创建 → /token → /introspect → active
- [ ] JWT 吊销流程：/token/revoke → inactive
- [ ] Opaque 全流程
- [ ] API Key 不受影响（回归）
