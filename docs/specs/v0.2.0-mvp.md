Pandaria api-gateway 不再使用 HMAC token，改为调 Aspectus `POST /introspect` 验证 API Key。交付完整的管理 API（Tenant/SA/APIKey CRUD）+ 审计日志 + Redis 缓存。

## 3. `/introspect` 端点（P0）

### 3.1 请求

```
POST /introspect
Authorization: Bearer {service_token}
Content-Type: application/x-www-form-urlencoded

token={subject_token}&token_type_hint=api_key
```

- `token` — 必填。待验证的 API Key 原文（如 `pk_live_aBcDeFgHiJkLmNoPqRsT`）
- `token_type_hint` — 可选。v0.2.0 仅处理 `api_key`，但接受任意值（预留扩展）
- `Authorization` header — Service Token，认证调用方身份

### 3.2 验证流程

```
1. 提取 Authorization header → ServiceTokenVerifier.verify(service_token)
   ├── sha256(service_token) → Redis (TTL=60s)
   │   ├── 命中 → 返回 Project 身份 → 继续步骤 2
   │   └── 未命中 → PostgreSQL (service_tokens 表)
   │       ├── 找到 → 写 Redis → 返回 Project → 继续步骤 2
   │       └── 未找到 → 401 application/problem+json
   │
2. 提取 body 中的 token 参数 → ApiKeyVerifier.verify(token)
   ├── sha256(token) → Redis (TTL=min(剩余有效期/10, 300s))
   │   ├── 命中 → 返回缓存的 IntrospectResponse（检查 revoked_at / expires_at）
   │   │   ├── 有效 → 200 IntrospectResponse
   │   │   └── 过期/吊销 → 200 { active: false }
   │   └── 未命中 → PostgreSQL (api_keys 表)
   │       ├── 找到且有效 → 写 Redis 缓存 → 200 IntrospectResponse
   │       ├── 找到但吊销/过期 → 200 { active: false }
   │       └── 未找到 → 200 { active: false }
```

### 3.3 响应

**有效 token**：
```json
HTTP 200
Content-Type: application/json

{
  "active": true,
  "tenant_id": "org-acme",
  "user_id": "sa-abc123def",
  "identity_type": "service_account",
  "client_id": "pandaria",
  "scope": "pandaria:session:create pandaria:session:read",
  "token_type": "Bearer",
  "exp": 1735689600
}
```

**无效/过期/吊销 token**：
```json
HTTP 200
Content-Type: application/json

{ "active": false }
```

**无效 Service Token**：
```json
HTTP 401
Content-Type: application/problem+json

{
  "type": "https://aspectus.dev/errors/unauthorized",
  "title": "Unauthorized",
  "status": 401,
  "detail": "Invalid or missing Service Token",
  "instance": "/introspect"
}
```

### 3.4 字段映射

| 响应字段 | 来源 | 说明 |
|---------|------|------|
| `active` | — | `true` / `false` |
| `tenant_id` | `api_keys.tenant_id` | |
| `user_id` | `api_keys.service_account_id` | v0.2.0 只有 SA，字段名保持 `user_id` 与 RFC 7662 兼容 |
| `identity_type` | 固定 `"service_account"` | |
| `client_id` | `api_keys.project` | Display 格式（小写） |
| `scope` | `api_keys.scopes` | `text[]` → 空格分隔字符串 |
| `token_type` | 固定 `"Bearer"` | |
| `exp` | `api_keys.expires_at` | Unix 时间戳。null → 不返回此字段 |

### 3.5 缓存策略

| 缓存对象 | Key | TTL | 存储内容 |
|---------|-----|-----|---------|
| API Key 自省结果 | `introspect:{sha256(token)}` | `min(剩余有效期秒数 / 10, 300)` | 完整 `IntrospectResponse` (JSON) |
| Service Token | `svc_token:{sha256(token)}` | 60s | Project 名称 (String) |

**缓存失效时机**：
- API Key 被吊销 → `DELETE` Redis key
- Service Token 被轮转 → 更新 DB → 等待 60s 自然过期

## 4. 管理 API（P1）

所有管理 API 需要认证。v0.2.0 使用 **管理 API Key** 认证（通过 `Authorization: Bearer {mgmt_key}` header），与 Service Token 区分。

> **v0.2.0 简化**：管理 API 的认证复用 Service Token 机制，但检查 `project == pandaria` 或引入一个 `mgmt` 伪 project。实际实现：创建 seed 数据时写入一个管理用 Service Token。

### 4.1 端点列表

| 方法 | 路径 | 请求体 | 响应 | 审计事件 |
|------|------|--------|------|---------|
| POST | `/tenants` | `{ "name": "org-acme" }` | `201 Tenant` | `tenant.created` |
| GET | `/tenants/{id}` | — | `200 Tenant` | — |
| POST | `/service-accounts` | `{ "tenant_id", "label", "description?" }` | `201 ServiceAccount` | `service_account.created` |

```
Response 200:
[
  { "id": "sa_...", "tenant_id": "...", "label": "CI Pipeline", ... }

```

#### `POST /api-keys`

```
Request:
{
  "service_account_id": "sa_xYzAbCdEfGhIjKlMnOpQr",
  "project": "pandaria",
  "scopes": ["pandaria:session:create", "pandaria:session:read"],
  "expires_at": null  // 可选，ISO 8601
}

Response 201:
{
  "id": "pk_aBcDeFgHiJkLmNoPqRsT",
  "key": "pk_live_aBcDeFgHiJkLmNoPqRsT",
  "key_prefix": "pk_live_aBcDeFgH",
  "project": "pandaria",
  "scopes": ["pandaria:session:create", "pandaria:session:read"],
  "expires_at": null
}

审计: api_key.created { api_key_id, service_account_id, project, scopes }
```

> ⚠️ `key` 字段仅在创建响应中返回。后续任何 API 都不返回原文。

**校验**：
- `service_account_id` 必须存在
- `project` 必须是合法枚举值
- `scopes` 数组可为空（v0.2.0 不做 scope 合法性校验——v0.3.0 加入）
- `expires_at` 若提供，必须 > now

**生成逻辑**（`ApiKeyCreator.create`）：
```
1. 生成 KSUID → id (e.g. "pk_aBcDeFgHiJkLmNoPqRsT")
2. 生成 32-byte 随机字节 → raw_key
3. 格式化为 "pk_live_{base62(raw_key)}"
4. key_hash = sha256(raw_key)
5. key_prefix = "pk_live_" + key 的第 9-16 字符
6. INSERT INTO api_keys (...) RETURNING id
7. 返回 CreatedApiKey { id, key: formatted, key_prefix, ... }
```

#### `GET /api-keys?service_account_id=...`

```
Response 200:
[
  {
    "id": "pk_...",
    "service_account_id": "sa_...",
    "project": "pandaria",
    "key_prefix": "pk_live_aBcDeFgH",
    "scopes": ["pandaria:session:create"],
    "expires_at": null,
    "revoked_at": null,
    "created_at": "2026-05-31T12:00:00Z"
  }
]
```

不返回 `key_hash` 或 `key` 原文。

#### `DELETE /api-keys/{id}`

```
Response 204 (No Content)

审计: api_key.revoked { api_key_id, revoked_at }
```

**操作**：
1. `UPDATE api_keys SET revoked_at = now() WHERE id = $1`
2. `DELETE` Redis key `introspect:{sha256(token)}`（如果存在）

已吊销的 Key 再次吊销 → 幂等，仍返回 204。

## 6. 认证实现 (`aspectus-auth`)

### 6.1 ApiKeyCreator

```rust
pub struct ApiKeyCreator {
    store: Arc<dyn ApiKeyStore>,
}

impl ApiKeyCreator {
    pub fn new(store: Arc<dyn ApiKeyStore>) -> Self { Self { store } }

    pub async fn create(
        &self,
        tenant_id: &str,
        service_account_id: &str,
        project: Project,
        scopes: Vec<String>,
        expires_at: Option<DateTime<Utc>>,
    ) -> Result<CreatedApiKey, CoreError> {
        // 1. Generate KSUID for the key ID (pk_ prefix for DB)
        let id = format!("pk_{}", ksuid::Ksuid::new());
        
        // 2. Generate 32 random bytes
        let mut raw = [0u8; 32];
        getrandom::getrandom(&mut raw)
            .map_err(|e| CoreError::Internal(format!("RNG error: {e}")))?;
        
        // 3. Format: "pk_live_" + base62(raw)
        let key = format!("pk_live_{}", base62_encode(&raw));
        
        // 4. key_hash = sha256(raw bytes, not the formatted string)
        use sha2::{Sha256, Digest};
        let key_hash = hex::encode(Sha256::digest(&raw));
        
        // 5. key_prefix for UI display
        let key_prefix = key[..17].to_string(); // "pk_live_" + 8 chars
        
        // 6. Store
        let db_key = self.store.insert(
            &id, tenant_id, service_account_id, project,
            &key_hash, &key_prefix, &scopes, expires_at,
        ).await?;
        
        Ok(CreatedApiKey {
            id,
            key,
            key_prefix,
            project,
            scopes,
            expires_at,
        })
    }
}
```

### 6.2 ApiKeyVerifier

**Helper functions** (defined in `aspectus-auth/src/verifier.rs`):

```rust
use sha2::{Sha256, Digest};

/// Extract raw bytes from a "pk_live_{hex}" formatted key.
fn extract_raw_from_key(token: &str) -> Option<Vec<u8>> {
    let encoded = token.strip_prefix("pk_live_")?;
    hex::decode(encoded).ok()
}

/// SHA256 hash as hex string.
fn sha256_hex(data: &[u8]) -> String {
    hex::encode(Sha256::digest(data))
}

/// Build an IntrospectResponse from a valid ApiKey row.
fn build_response(api_key: &ApiKey) -> IntrospectResponse {
    IntrospectResponse {
        active: true,
        tenant_id: Some(api_key.tenant_id.clone()),
        user_id: Some(api_key.service_account_id.clone()),
        identity_type: Some(IdentityType::ServiceAccount),
        client_id: Some(api_key.project.to_string()),
        scope: Some(api_key.scopes.join(" ")),
        token_type: Some("Bearer".into()),
        exp: api_key.expires_at.map(|dt| dt.timestamp()),
        quotas: None,
    }
}

/// Compute Redis cache TTL: min(remaining_seconds / 10, 300).
fn compute_cache_ttl(expires_at: Option<DateTime<Utc>>) -> u64 {
    match expires_at {
        Some(exp) => {
            let remaining = (exp - Utc::now()).num_seconds().max(0) as u64;
            (remaining / 10).min(300)
        }
        None => 300,
    }
}
```

**Verifier struct**:
    store: Arc<dyn ApiKeyStore>,
    cache: RedisCache,
}

impl ApiKeyVerifier {
    pub fn new(store: Arc<dyn ApiKeyStore>, cache: RedisCache) -> Self {
        Self { store, cache }
    }

    pub async fn verify(&self, token: &str) -> IntrospectResponse {
        // 1. Extract raw bytes from formatted key
        let raw = match extract_raw_from_key(token) {
            Some(r) => r,
            None => return IntrospectResponse::inactive(),
        };
        
        // 2. key_hash = sha256(raw)
        let key_hash = sha256_hex(&raw);
        
        // 3. Redis cache lookup
        let cache_key = format!("introspect:{key_hash}");
        if let Some(cached) = self.cache.get_json::<IntrospectResponse>(&cache_key).await {
            // Re-check expiry (revocation handled via cache key deletion on revoke)
            if let Some(exp) = cached.exp {
                if exp < Utc::now().timestamp() {
                    return IntrospectResponse::inactive();
                }
            }
            return cached;
        }
        
        // 4. PostgreSQL fallback
        match self.store.find_by_hash(&key_hash).await {
            Ok(Some(api_key)) => {
                if api_key.revoked_at.is_some() {
                    return IntrospectResponse::inactive();
                }
                if let Some(exp) = api_key.expires_at {
                    if exp < Utc::now() {
                        return IntrospectResponse::inactive();
                    }
                }
                
                let response = build_response(&api_key);
                
                // Cache it
                let ttl = compute_cache_ttl(api_key.expires_at);
                self.cache.set_json(&cache_key, &response, ttl).await;
                
                response
            }
            Ok(None) => IntrospectResponse::inactive(),
            Err(_) => IntrospectResponse::inactive(), // DB error → safe fallback
        }
    }
}
```

### 6.3 ServiceTokenVerifier

```rust
pub struct ServiceTokenVerifier {
    store: Arc<dyn ServiceTokenStore>,
    cache: RedisCache,
}

impl ServiceTokenVerifier {
    pub fn new(store: Arc<dyn ServiceTokenStore>, cache: RedisCache) -> Self {
        Self { store, cache }
    }

    pub async fn verify(&self, token: &str) -> Option<Project> {
        let token_hash = sha256_hex(token.as_bytes());
        let cache_key = format!("svc_token:{token_hash}");
        
        // Redis (TTL=60s)
        if let Some(project_str) = self.cache.get::<String>(&cache_key).await {
            return project_str.parse().ok();
        }
        
        // PostgreSQL
        match self.store.find_by_hash(&token_hash).await {
            Ok(Some(project)) => {
                self.cache.set(&cache_key, &project.to_string(), 60).await;
                Some(project)
            }
            _ => None,
        }
    }
}
```

// crates/aspectus-auth/src/cache.rs

pub struct RedisCache {
    conn: redis::aio::ConnectionManager,
}

impl RedisCache {
    pub fn new(client: redis::Client) -> Self {
        Self { conn: client.into() }
    }

    pub async fn get_json<T: DeserializeOwned>(&self, key: &str) -> Option<T> {
        let raw: Option<String> = self.conn.clone().get(key).await.ok()?;
        raw.and_then(|s| serde_json::from_str(&s).ok())
    }

    pub async fn set_json<T: Serialize>(
        &self, key: &str, value: &T, ttl_secs: u64,
    ) {
        let json = serde_json::to_string(value).unwrap();
        let _: Result<(), _> = self.conn.clone().set_ex(key, json, ttl_secs).await;
    }

    pub async fn get<T: FromRedisValue>(&self, key: &str) -> Option<T> {
        self.conn.clone().get(key).await.ok()
    }

    pub async fn set<T: ToRedisArgs + Send + Sync>(
        &self, key: &str, value: &T, ttl_secs: u64,
    ) {
        let _: Result<(), _> = self.conn.clone().set_ex(key, value, ttl_secs).await;
    }

    pub async fn del(&self, key: &str) {
        let _: Result<(), _> = self.conn.clone().del(key).await;
    }
}

## 7. HTTP Middleware & 路由 (`aspectus-server`)

### 7.1 Service Token Middleware

```rust
// crates/aspectus-server/src/middleware/auth.rs

use axum::{
    extract::Request,
    http::StatusCode,
    middleware::Next,
    response::{IntoResponse, Response},
};

pub async fn service_token_auth(
    request: Request,
    next: Next,
) -> Response {
    // Extract Authorization: Bearer {token}
    let auth_header = request
        .headers()
        .get(axum::http::header::AUTHORIZATION)
        .and_then(|v| v.to_str().ok())
        .and_then(|v| v.strip_prefix("Bearer "));

    let Some(token) = auth_header else {
        return ProblemDetails::unauthorized(
            "Missing Authorization header",
            request.uri().path(),
        ).into_response();
    };

    // Verify via ServiceTokenVerifier (injected via Extension)
    let verifier = request
        .extensions()
        .get::<Arc<ServiceTokenVerifier>>()
        .cloned();

    let Some(verifier) = verifier else {
        return ProblemDetails::internal_error("ServiceTokenVerifier not configured")
            .into_response();
    };

    match verifier.verify(token).await {
        Some(project) => {
            // Inject Project into request extensions for downstream handlers
            let mut request = request;
            request.extensions_mut().insert(project);
            next.run(request).await
        }
        None => ProblemDetails::unauthorized(
            "Invalid Service Token",
            request.uri().path(),
        ).into_response(),
    }
}
```

let mgmt = Router::new()
    .route("/tenants", post(tenants::create))
    .route("/tenants/{id}", get(tenants::get))
    .route("/service-accounts", post(service_accounts::create).get(service_accounts::list))
    .route("/api-keys", post(api_keys::create).get(api_keys::list))
    .route("/api-keys/{id}", delete(api_keys::revoke))
    .layer(middleware::from_fn(middleware::auth::service_token_auth));

let app = Router::new()
    // P0: Token introspection (Service Token auth)
    .route("/introspect", post(introspect::handle))
    .route_layer(middleware::from_fn(middleware::auth::service_token_auth))

    // P1: Management API
    .merge(mgmt)

    // Health
    .route("/health", get(|| async { "ok" }))

    // Layers
    .layer(CorsLayer::permissive())
    .layer(TraceLayer::new_for_http());
```

> **v0.2.0 简化**：管理 API 和 `/introspect` 共用同一个 Service Token middleware。生产环境建议管理 API 使用独立的 mgmt token，但 MVP 阶段不分离。

}

        Ok(None) => ProblemDetails::not_found(format!("Tenant {id} not found")).into_response(),
        Err(e) => ProblemDetails::internal_error(e.to_string()).into_response(),
    }
}

use aspectus_core::error::CoreError;

impl From<CoreError> for ProblemDetails {
    fn from(err: CoreError) -> Self {
        match &err {
            CoreError::NotFound { entity, id } => {
                Self::not_found(format!("{entity} not found: {id}"))
            }
            CoreError::Validation(msg) => {
                Self::validation_failed(msg.clone(), vec![])
            }
            _ => Self::internal_error(err.to_string()),
        }
    }
}

### 8.1 记录时机

| 事件 | 触发点 | actor_id | actor_type | metadata |
|------|--------|----------|------------|----------|

| `api_key.revoked` | DELETE /api-keys/{id} | `"mgmt"` | `service_account` | `{ "revoked_at" }` |
| `token.introspected` | POST /introspect | Service Token project | `service_account` | 仅采样（每 1000 次 1 次或仅 `active=false`） |

### 8.2 审计日志写入

所有 handler 在操作成功后调用 `AuditLogStore.append()`。写入失败不应阻塞主流程——记录 tracing error 但继续。

```rust
// 辅助函数
async fn audit<S: AuditLogStore>(
    store: &S,
    tenant_id: &str,
    actor_id: &str,
    actor_type: IdentityType,
    action: &str,
    target_type: &str,
    target_id: &str,
    metadata: serde_json::Value,
) {
    let entry = AuditLog {
        id: Ksuid::new().to_string(),
        tenant_id: tenant_id.to_string(),
        actor_id: actor_id.to_string(),
        actor_type,
        action: action.to_string(),
        target_type: target_type.to_string(),
        target_id: target_id.to_string(),
        metadata,
        created_at: Utc::now(),
    };
    
    if let Err(e) = store.append(entry).await {
        tracing::error!(error = %e, "Failed to write audit log");
    }
}
```

## 9. PostgreSQL Store 实现

### 9.1 PgTenantStore

```rust
#[async_trait]
impl TenantStore for PgTenantStore {
    async fn create(&self, name: &str) -> Result<Tenant, CoreError> {
        let id = Ksuid::new().to_string();
        sqlx::query_as::<_, Tenant>(
            "INSERT INTO tenants (id, name) VALUES ($1, $2) RETURNING *"
        )
        .bind(&id)
        .bind(name)
        .fetch_one(&self.pool)
        .await
        .map_err(|e| CoreError::Internal(e.to_string()))
    }

    async fn get_by_id(&self, id: &str) -> Result<Option<Tenant>, CoreError> {
        sqlx::query_as::<_, Tenant>("SELECT * FROM tenants WHERE id = $1")
            .bind(id)
            .fetch_optional(&self.pool)
            .await
            .map_err(|e| CoreError::Internal(e.to_string()))
    }
}
```

其余 Store 实现模式相同（`PgServiceAccountStore`, `PgApiKeyStore`, `PgAuditLogStore`, `PgServiceTokenStore`），参考上述模板。

### 9.2 ApiKeyStore 特殊查询

```rust
async fn find_by_hash(&self, key_hash: &str) -> Result<Option<ApiKey>, CoreError> {
    sqlx::query_as::<_, ApiKey>(
        "SELECT * FROM api_keys WHERE key_hash = $1"
    )
    .bind(key_hash)
    .fetch_optional(&self.pool)
    .await
    .map_err(|e| CoreError::Internal(e.to_string()))
}

async fn revoke(&self, id: &str) -> Result<bool, CoreError> {
    let result = sqlx::query(
        "UPDATE api_keys SET revoked_at = now() WHERE id = $1 AND revoked_at IS NULL"
    )
    .bind(id)
    .execute(&self.pool)
    .await
    .map_err(|e| CoreError::Internal(e.to_string()))?;
    
    Ok(result.rows_affected() > 0)
}
```
