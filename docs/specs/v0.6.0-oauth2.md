# v0.6.0 — OAuth2 Authorization Code Flow 技术规格

> 版本：v0.6.0
> 状态：已冻结
> 前置依赖：v0.5.0（User + Role + PasswordHasher + JWT）

---

## 1. 目标

用户可通过 OAuth2 Authorization Code flow 登录获取 JWT access token。复用 v0.4 JWT 和 v0.5 PasswordHasher。v0.6 仅实现核心 flow，不做 refresh token、不做 login UI（各项目自建）。

---

## 2. 端点

| 方法 | 路径 | 认证 | 说明 |
|------|------|:--:|------|
| POST | `/authorize` | 无 | 验证 email+password，返回 authorization code |
| POST | `/token` | 无 | 扩展 `grant_type=authorization_code`，返回 JWT |

---

## 3. Flow

```
1. 项目自建 login 页面，收集 email + password
2. POST /authorize { email, password, client_id, redirect_uri }
   → 验证密码 → 生成 code → 返回 { code, redirect_uri }
3. 项目用 code 调 POST /token
   { grant_type: "authorization_code", code, client_id }
   → 验证 code → 返回 { access_token (JWT), token_format: "jwt", expires_in }
4. 项目用 JWT 调 /introspect → active: true, identity_type: "user"
```

---

## 4. Authorization Code 存储

新建 `authorization_codes` 表：

```sql
CREATE TABLE authorization_codes (
    code VARCHAR(64) PRIMARY KEY,
    user_id VARCHAR(21) NOT NULL REFERENCES users(id),
    client_id VARCHAR(21) NOT NULL,
    redirect_uri TEXT NOT NULL,
    expires_at TIMESTAMPTZ NOT NULL,
    used BOOLEAN NOT NULL DEFAULT false
);
```

- code: sha256(random 32 bytes) hex → 64 chars
- TTL: 60 秒
- 一次性使用（used=true 后不可再用）

---

## 5. 实现要点

- `POST /authorize`: 验证 password → 生成 code → INSERT → 返回
- `POST /token` (authorization_code): 查 code → 验证未用/未过期 → 标记 used → JwtSigner.sign → 返回 JWT
- 复用 v0.4 JwtSigner，TTL=900s
- 复用 v0.5 PasswordHasher.verify
